# The Argent Order
## Portal Specification

Version: 2.0

---

# Purpose

The Portal is the center of The Argent Order ecosystem.

Discord is for brotherhood.

The Portal is for formation.

The Portal exists to help a man answer one question:

"Who am I becoming?"

Every feature should contribute to formation.

Every screen should drive action.

Every metric should encourage growth.

The Portal is not social media.

The Portal is not entertainment.

The Portal is a formation operating system.

---

# Core Philosophy

The Portal exists to create:

Clarity

↓

Action

↓

Consistency

↓

Growth

↓

Transformation

Everything must support this process.

# PORTAL PRODUCT SPECIFICATION

Version: 2.0

Status: Canonical Product Architecture

---

## Product Philosophy

The Portal should not feel like:

* A social media platform
* A productivity app
* A generic habit tracker
* A course platform

It should feel like:

```
A Catholic formation command centre
```

A place where a man enters, sees his mission, executes his responsibilities, and tracks his growth.

---

## Core User Loop

```
Open Portal → See Mission → Complete Daily Rule → Progress Campaign → Report Accountability → Review Growth → Increase Responsibility
```

---

## User Journey

### Stage 1 — New Member (Initiate)

User completes: Account setup, Profile creation, Rules acceptance, Rule Of Life creation, First campaign selection

### Stage 2 — Active Brother

Access: Dashboard, Campaigns, Tracking, Brotherhood systems

### Stage 3 — Leader (Captain+)

Access: Pod management, Member accountability, Leadership tools

---

## MAIN DASHBOARD

### Daily Mission

Displays: Prayer task, Discipline task, Building task, Reflection task

### Current Campaign

Displays: Campaign name, Progress, Current day, Completion rate

### Rule Of Life Status

Displays: Current Rule, Streak, Weak areas

### Brotherhood Status

Displays: Pod, Check-in status, Upcoming meetings

### Rank Progress

Displays: Current rank, Next rank, Requirements

---

## RULE OF LIFE BUILDER

Flow: Choose Template → Customize → Set Commitments → Activate

Templates: Universal Rule, Student Rule, Builder Rule, Entrepreneur Rule, Husband Rule, Father Rule

Custom Categories: Faith (Prayer, Scripture, Mass, Rosary), Discipline (Exercise, Sleep, Fasting, Digital limits), Brotherhood (Pod attendance, Service), Building (Deep work, Projects, Learning), Truth (Reflection, Weekly review)

---

## DAILY CHECK-IN SYSTEM

Daily check-in states: Completed / Missed / Needs Review

Not designed for perfection. Designed for honesty.

---

## FAILURE SYSTEM

When a task is missed:

User records: Why, What happened, Correction

The system asks: "What will change tomorrow?"

---

## REVIEW SYSTEM

Daily Review: What did I complete? Where did I fail? What needs correction?

Weekly Review: What improved? What weakened? What is the next focus?

Monthly Review: Who am I becoming? What must change?

---

## BROTHERHOOD SYSTEM

Portal connection to Discord.

Features:

* Pod Dashboard (Members, Activity, Check-ins)
* Accountability Requests
* Pod Reviews (Weekly: Wins, Struggles, Goals)

---

## CERTIFICATION SYSTEM

Certification Page shows: Certifications, Progress, Requirements, History

---

## LEADERSHIP DASHBOARD (Captain+)

Features: Pod Overview, Member Support, Reports

---

## PROFILE SYSTEM

Displays: Name, Rank, Certifications, Campaign history, Builder projects

Focus: Growth evidence. Avoid: Vanity metrics.

---

## GAMIFICATION PHILOSOPHY

Gamification should reinforce virtue. Never addiction.

Reward: Consistency, Completion, Service, Leadership

Avoid: Endless points, Attention farming, Competition without purpose

---

## POINT SYSTEM

### Formation Points

Earned by: Completing Rule tasks, Campaign progress, Helping brothers

### Builder Points

Earned by: Projects, Contributions

### Service Points

Earned by: Leadership, Helping others

---

## NOTIFICATIONS

Daily: Mission reminder

Weekly: Review reminder

Campaign: Progress reminder

Support discipline. Not create dependency.

---

## ADMIN SYSTEM

Admins manage: Campaigns, Certifications, Users, Analytics, Content

---

## TECH REQUIREMENTS

Frontend: Next.js, TypeScript, React

Backend: API routes, Services, Database layer

Database stores: Users, Rules, Tasks, Campaigns, Progress, Certifications, Reviews

---

## SECURITY

Requirements: Authentication, Role permissions, Data protection, Audit logging
