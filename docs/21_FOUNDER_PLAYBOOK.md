# The Argent Order

## Founder Playbook

Version: 2.0

Status: Canonical Execution Manual for Founder

---

# Purpose

This document defines how the founder operates inside The Argent Order.

Not as a content creator.

Not as a community manager.

But as a system builder.

The founder’s job is to:

```text id="purpose"
Design systems

Build leaders

Remove bottlenecks

Protect mission

Drive alignment
```

---

# Core Founder Rule

If the system depends on you daily, you have failed as a founder.

---

# Founder's Role Evolution

## Phase 1 — Builder

You design everything.

You define:

* Vision
* Structure
* Systems
* Culture

---

## Phase 2 — Operator

You refine systems.

You fix bottlenecks.

You delegate execution.

---

## Phase 3 — Architect

You stop doing tasks.

You design outcomes.

You only adjust systems and leadership.

---

## Phase 4 — Steward

You protect mission integrity.

You ensure continuity.

You focus on long-term direction.

# DECISION FRAMEWORK

Every decision must pass:

---

## Faith Test

Does this align with Catholic formation?

---

## Discipline Test

Does this increase responsibility?

---

## Brotherhood Test

Does this strengthen men?

---

## Builder Test

Does this increase output?

---

## Truth Test

Does this reflect reality?

---

If it fails 2+ tests:

Do not implement.

# BOTTLENECK RULE

The system always fails at the weakest point.

Common bottlenecks:

* Weak onboarding
* Weak leadership
* Weak accountability
* Weak campaigns
* Weak content clarity

---

Founder responsibility:

Fix bottlenecks only.

Ignore noise.

---

# DELEGATION RULE

If someone else can do it at 80% quality:

Delegate it.

If someone else can do it at 100% quality:

Remove yourself completely.
